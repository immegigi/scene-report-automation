# Codex 3차 명령어: 리포트 에디터 + PDF 생성

기존 관리자 MVP에 리포트 에디터, PDF 미리보기, PDF 다운로드 기능을 구현해줘.

## 목표

AI가 생성한 5페이지 리포트 원고를 운영자가 수정한 뒤,
고객에게 보낼 수 있는 PDF로 생성하고 다운로드할 수 있게 만든다.

초기 버전에서는 이메일 자동 발송은 하지 않는다.
이메일 제목과 본문은 생성 후 복사할 수 있게 만든다.

---

## 리포트 PDF 구조

총 5페이지다.

1p. 당신의 장면
2p. 관계 인지 루프
3p. 내 마음이 반복해서 만드는 관계 장면
4p. 다음 대화 문장
5p. 다음 장면 기록지

---

## 리포트 에디터

리포트 에디터에서 다음 항목을 수정 가능하게 만든다.

- email_subject
- email_body
- pdf_page_1
- pdf_page_2
- pdf_page_3
- pdf_page_4
- pdf_page_5
- review_status

각 페이지는 textarea 또는 rich text editor로 수정 가능해야 한다.

---

## PDF 디자인

PDF는 감성 편지와 전문 리포트의 중간 느낌이어야 한다.

디자인 기준:

- 웜 아이보리 배경
- 차콜 텍스트
- 로즈 브라운 포인트
- 과한 심리검사표 느낌 금지
- 치료센터 브로슈어 느낌 금지
- 모바일에서도 읽기 쉬운 여백
- 각 페이지 상단에 작은 섹션 타이틀
- 하단에 브랜드명: 날빛코칭

---

## PDF 구현 방식

권장 방식:

- HTML/CSS 템플릿 생성
- Playwright 또는 Puppeteer로 PDF 렌더링
- 생성된 PDF를 Supabase Storage에 저장
- reports.pdf_url에 저장
- 관리자 화면에서 다운로드 가능하게 표시

---

## PDF 템플릿 요구

각 페이지는 다음 구조를 가진다.

### Page 1

제목: 당신의 장면

내용:

- 고객 장면 요약
- 실제로 일어난 일
- 내가 붙인 해석
- 마음이 커진 지점

### Page 2

제목: 관계 인지 루프

내용:

- 자극
- 주의
- 해석
- 감정
- 반응

### Page 3

제목: 내 마음이 반복해서 만드는 관계 장면

내용:

- S-C-B 코드
- 패턴명
- 인지 필터 설명
- 대상관계적 해석
- 반복될 때의 관계 문제

### Page 4

제목: 다음 대화 문장

내용:

- 바로 보내지 않는 편이 좋은 말
- 부드러운 요청 문장
- 명확한 확인 문장
- 경계 문장

### Page 5

제목: 다음 장면 기록지

내용:

- 오늘의 장면
- 실제로 일어난 일
- 내가 붙인 해석
- 감정
- 반응
- 하고 싶었지만 못 한 말
- 다음에 바꿔볼 문장

마지막에는 다음 문장을 자연스럽게 포함한다.

“이번 리포트는 끝이 아니라 첫 번째 관계 장면 기록입니다.
비슷한 장면은 흘려보내지 말고 모아보는 편이 좋습니다.
모인 장면은 감정의 증거가 아니라 나를 이해하는 자료가 됩니다.”

---

## 이메일 본문 복사 기능

리포트 에디터 화면에 다음 버튼을 추가한다.

- 이메일 제목 복사
- 이메일 본문 복사
- PDF 다운로드

---

## 상태값 처리

PDF 생성 성공 시:

- review_status = approved
- submission.status = pdf_generated
- reports.pdf_url 업데이트

PDF 생성 실패 시:

- 오류 메시지 표시
- status 변경하지 않음
